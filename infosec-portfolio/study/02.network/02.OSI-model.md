# OSI 7 Layer

## 개요
OSI 7계층(Open Systems Interconnection)은 네트워크 통신 과정을 7단계로 나누어
각 계층의 역할을 명확히 하기 위한 표준 모델이다.

---

## 계층 구조

| 계층 | 이름 | 역할 |
| 7 | Application | 사용자와 직접 상호작용 |
| 6 | Presentation | 데이터 변환, 암호화 |
| 5 | Session | 연결 관리 |
| 4 | Transport | 데이터 전송, 오류 제어 |
| 3 | Network | 경로 설정, IP |
| 2 | Data Link | MAC 주소, 오류 검출 |
| 1 | Physical | 전기 신호 전송 |

---

## 🔍 계층별 설명

### 7️ Application
- 사용자 인터페이스 제공
- HTTP, FTP, SMTP

---

### 6️ Presentation
- 데이터 형식 변환
- 암호화 / 복호화

---

### 5️ Session
- 통신 세션 생성 및 종료
- 연결 유지

---

### 4️ Transport
- 데이터 분할 및 재조립
- TCP / UDP

---

### 3️ Network
- IP 주소 기반 라우팅
- 경로 선택

---

### 2️ Data Link
- MAC 주소 기반 통신
- 오류 검출

---

### 1️ Physical
- 전기적 신호 전송
- 케이블, 허브

---

## 💡 특징
- 계층별 독립적인 구조
- 문제 발생 시 계층별 분석 가능
- 표준화된 네트워크 모델

---

## 🔗 TCP/IP와 비교

| OSI | TCP/IP |

| Application | Application |
| Presentation | Application |
| Session | Application |
| Transport | Transport |
| Network | Internet |
| Data Link | Network Access |
| Physical | Network Access |

---

## 💭 느낀 점
처음에는 계층이 많아 복잡하게 느껴졌지만,
각 계층의 역할을 나누어 생각하니 네트워크 구조를 이해하는 데 도움이 되었다.
