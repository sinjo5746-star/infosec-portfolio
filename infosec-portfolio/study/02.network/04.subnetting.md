## 개요
Subnetting은 하나의 네트워크를 여러 개의 작은 네트워크로 나누는 기술이다.  
IP 주소를 효율적으로 사용하고, 네트워크를 체계적으로 관리하기 위해 사용된다.

---

## 기본 개념

### IP 구조
IP 주소는 **네트워크 부분 + 호스트 부분**으로 구성된다.
예: 192.168.1.0 /24
- /24 → 앞 24비트는 네트워크
- 나머지 8비트 → 호스트

---

## 호스트 수 계산
:contentReference[oaicite:0]{index=0}
- n = 호스트 비트 수
- -2 → 네트워크 주소 + 브로드캐스트 주소 제외

---

## 서브넷 마스크

| CIDR | 서브넷 마스크 |

| /24 | 255.255.255.0 |
| /25 | 255.255.255.128 |
| /26 | 255.255.255.192 |
| /27 | 255.255.255.224 |

---

## Subnetting 과정

1. 필요한 호스트 수 확인
2. 호스트 비트 수 계산
3. 서브넷 마스크 결정
4. 네트워크 분할

---

## 예제 문제

### 문제 01.

200.196.244.0/24
영업부: 100대
총무부: 50대
자재부: 25대
관리부: 25대

-> 풀이 과정
### 필요한 호스트 수 확인
- 100 → 최소 128 필요 → /25
- 50 → 최소 64 필요 → /26
- 25 → 최소 32 필요 → /27
- 25 → 최소 32 필요 → /27

### 네트워크 할당 (큰 것부터)
- 영업부 (/25)
200.196.244.0 ~ 200.196.244.127
- 총무부 (/26)
200.196.244.128 ~ 200.196.244.191
- 자재부 (/27)
200.196.244.192 ~ 200.196.244.223
- 관리부 (/27)
200.196.244.224 ~ 200.196.244.255

---

## 📊 결과 정리

| 부서 | 네트워크 | 범위 |

| 영업부 | /25 | 0 ~ 127 |
| 총무부 | /26 | 128 ~ 191 |
| 자재부 | /27 | 192 ~ 223 |
| 관리부 | /27 | 224 ~ 255 |

---

## 💡 핵심 포인트
- 큰 네트워크부터 할당 (VLSM)
- 2의 제곱으로 계산
- 네트워크/브로드캐스트 제외

---

## 💭 느낀 점
처음에는 계산이 어렵게 느껴졌지만,  
큰 네트워크부터 나누는 방식으로 접근하니 이해가 쉬워졌다.

---
