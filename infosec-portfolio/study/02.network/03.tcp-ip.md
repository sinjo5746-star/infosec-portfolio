## 개요
TCP/IP 모델은 인터넷에서 사용되는 대표적인 네트워크 프로토콜 집합으로,
데이터 통신 과정을 계층적으로 나누어 정의한 모델이다.

---

## 계층 구조

| 계층 | 이름 | 역할 |

| 4 | Application | 사용자 서비스 제공 |
| 3 | Transport | 데이터 전송 |
| 2 | Internet | IP 주소 기반 라우팅 |
| 1 | Network Access | 물리적 데이터 전송 |

---

## 계층별 설명

### 4️ Application
- 사용자와 직접 상호작용
- HTTP, FTP, DNS

---

### 3️ Transport
- 데이터 전송 및 제어
- TCP / UDP

---

### 2️ Internet
- IP 주소 기반 통신
- 라우팅 수행

---

### 1️ Network Access
- 실제 데이터 전송
- MAC 주소 사용

---

## 🔗 OSI 7 Layer와 비교

| TCP/IP | OSI |

| Application | Application + Presentation + Session |
| Transport | Transport |
| Internet | Network |
| Network Access | Data Link + Physical |

---

## 💡 TCP vs UDP

| 구분 | TCP | UDP |

| 연결 방식 | 연결형 | 비연결형 |
| 신뢰성 | 높음 | 낮음 |
| 속도 | 느림 | 빠름 |
| 예시 | HTTP, FTP | DNS, 스트리밍 |

---

## 💡 특징
- 인터넷 표준 프로토콜
- OSI보다 단순한 구조
- 실제 네트워크에서 사용

---

## 💭 느낀 점
OSI 모델보다 구조가 단순하여 실제 네트워크 동작을 이해하는 데 더 직관적이었다.

---
